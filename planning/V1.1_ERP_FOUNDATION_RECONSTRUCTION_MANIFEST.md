## Pre-Merge Hardening Boundary

The foundation now performs structural validation only:
- every non-empty canonical target must exist in the approved schema for its sheet;
- duplicate canonical targets within one sheet are rejected;
- empty custom-template mappings remain valid;
- XLSX packages containing macros, external workbook links, connection definitions,
  query tables or external-data definitions are rejected;
- repository fixtures are checked offline for those package features and
  credential-like connection-string markers.

These controls do not transform data, normalize records, execute mappings or
prove that visible workbook values contain no real personal, supplier-confidential
or commercial information.

## Remaining Manual Gate

Before merge, the owner must open all three workbook fixtures and inspect visible
values for:
- real names or personal data;
- real supplier-identifying information;
- real commercial prices or terms;
- credentials or confidential notes.

Automated package checks are not a substitute for this visible-data review.

## Frozen Portfolio Controls

Portfolio Edition v1.0.1 application logic, formulas, schemas, rankings,
thresholds, recommendation eligibility, approval controls, datasets, workflows,
dependencies, release metadata, tags, releases and deployments must remain
unchanged.
