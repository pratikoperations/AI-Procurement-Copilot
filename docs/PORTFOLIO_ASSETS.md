# Portfolio Assets

## Project Positioning

AI Procurement Copilot is a transparent, rule-guided procurement decision-support application for packaging sourcing. It converts RFQ data into should-cost, risk-adjusted TCO, supplier scoring, scenario analysis, allocation, negotiation, and executive recommendations.

## Resume Bullets

- Built an AI-enabled Procurement Copilot in Python and Streamlit that converts packaging RFQ data into transparent supplier recommendations using should-cost, TCO, risk, ESG, performance, and allocation logic.
- Designed an auditable best-value sourcing engine that separates lowest price from true procurement value by incorporating MOQ, payment terms, lead time, freight, working capital, quality, and continuity risk.
- Developed scenario simulation, negotiation planning, executive memo generation, supplier communication drafts, and downloadable decision packages for procurement leadership use.
- Added automated regression testing, Streamlit smoke testing, RFQ validation, GitHub Actions CI, and release governance to improve reliability and maintainability.

## LinkedIn Project Description

Built an AI Procurement Copilot focused on packaging sourcing and explainable decision support. The application analyzes RFQs, calculates packaging should-cost, adjusts supplier quotations for total cost of ownership and risk, evaluates ESG and performance, recommends supplier allocation, runs stress scenarios, generates negotiation strategies, and produces executive-ready outputs.

The core design principle is transparent and auditable procurement logic. AI assists explanation and communication, while procurement retains control of commercial assumptions, scoring, governance, and award decisions.

## Interview Value Statement

“I built a procurement decision-support product that goes beyond dashboards. It translates RFQ and supplier data into an explainable recommendation, shows why lowest price may not equal best value, quantifies risk and TCO, recommends allocation, and generates executive and supplier-facing outputs.”
